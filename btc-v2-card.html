🟢 진입 카드 — STAGE 0~3

자동화 계산기 (스프레드시트) 준비 후 사용. 카드에는 결과값만 입력.

⛔ STAGE 0 — 시장 차단 (10초)

5묶음 중 하나라도 ❌ → 즉시 차트 닫고 관망.

□ 1. 신호 부족
   - 거래량 4H 5MA 1.3x 미만
   - RR < 1.5
   - Stop 위치 구조 불명확

□ 2. 시장 환경 부적합
   - 시나리오 C 활성 (직전 핵심 지지 일봉 종가 이탈)
   - 거래량 고갈 무진입 룰 활성
   - 20일선 이격 +7% 이상

□ 3. 진입 근거 불완전
   - 일봉 종가 진입 근거 레벨 위 안착 미확인
   - A-2 진입 타이밍 룰 위반 (시간 +2캔들 / 가격 +1.5% / RR 2.0)

□ 4. 주문 준비 불가
   - 사이즈 정의 (마진 / 노출 / BTC 수량) 사전 확정 불가
   - TP0 + Stop 즉시 세팅 불가

□ 5. 리스크 초과
   - 예상 시드 손실 -10% 초과
   - B급인데 TP1 RR < 2.0

→ 5묶음 모두 ✅ → STAGE 1
→ 본문 11개 상세는 룰북 참조

🟢 STAGE 1 — 진입 분류 (15~30초)

[1] 시나리오 확정
   □ A (돌파 후 눌림목)
   □ A-2 (강세 추종)
   □ B (지지 반등)

[2] 매트릭스 4요소 점수 (1~3점)
   □ 거래량 _점
   □ RSI _점
   □ 캔들 강도 _점
   □ 구조 정합성 _점
   합계: ___점

[3] 등급 결정
   □ 5점 이하: 진입 금지
   □ 6~7점: B급 (마진 30~40%, Stop -7% 이내)
   □ 8~10점: A급 (마진 45~55%, Stop -5% 이내)
   □ 11~12점: S급 (마진 60~70%, Stop -3% 이내) [80% 봉쇄]

[4] A-2 한정 추가 체크
   □ 트리거 캔들 마감 후 +2캔들(8h) 이내?
   □ 진입가 트리거 종가 대비 +1.5% 이내?
   □ TP1 RR ≥ 2.0 유지?

[5] 동적 하향 체크
   □ Stop 거리가 등급 한도 초과? → 한 단계 하향
   □ 강등 트리거 (다이버전스 / 윗꼬리 30%+ / 20일선 +5%~+7%)? → 한 단계 하향

→ 모든 체크 통과 → STAGE 2

🔵 STAGE 2 — 사이즈/Stop 계산 (20~30초)

자동화 계산기 사용 권장. 입력만 하고 결과 확인.

[입력]
시드 S = ___ USDT
등급 사이즈 % M = ___%
진입가 P = $___
Stop 가격 SL = $___
레버리지 = 3x (고정)
Lot Size = 0.001 BTC (주문 전 OKX 재확인)

[사이즈 계산 7단계]
1. 마진 의도 = S × M = ___ USDT
2. 노출 의도 = 마진 × 3 = ___ USDT
3. BTC 수량 (이론) = 노출 / P = 0.____ BTC
4. BTC 수량 (실제, Lot 내림) = 0.____ BTC ⭐
5. 실제 노출 = BTC × P = ___ USDT
6. 실제 마진 = 노출 / 3 = ___ USDT
7. 시드 사용률 = 마진 / S = ___%

[Stop 검증 4단계]
1. Stop 거리 % = (P-SL)/P = ___%
2. 예상 손실 = (P-SL) × BTC = ___ USDT
3. 예상 시드 손실률 = 손실/S = ___%
4. 검증
   □ Stop 거리 ≤ 등급 한도? (S -3% / A -5% / B -7%)
   □ 시드 손실률 < 10%?

→ 둘 다 ✅ → STAGE 3

🔴 STAGE 3 — 주문 실행 + 시각적 대조 (15~30초)

OKX Amount = 노출(Notional)이지 마진 ❌

권장: BTC Size 직접 입력

[OKX 입력]
□ Size 필드에 BTC 수량 직접 입력 (권장)
   → STAGE 2 단계 4번 값 그대로

또는 □ Amount 필드에 노출 USDT 입력
   → STAGE 2 단계 5번 값
   🚨 절대 금지: Amount에 "마진" USDT 입력

[시각적 대조 5체크 — 모두 ✅ 시 주문 제출]
□ 계산표 BTC 수량 = OKX Size 입력값
□ 계산표 실제 노출 = OKX 표시 노출
□ 계산표 실제 마진 = OKX 표시 마진
□ Stop 주문 수량 = 실제 포지션 수량
□ TP0 주문 수량 = 계획 비중 (S/A 20% / B 30%)

[STAGE 2 의무 (체결 직후 30초 이내)]
□ TP0 지정가 세팅
□ Stop-Market 세팅 (구조 기반)
□ TP1 Limit Close 등록 (RR ≥ 2.0)

→ 모든 체크 ✅ → 주문 제출
→ 하나라도 ❌ → 주문 취소 후 재계산

🟡 운영 카드 — 3-Layer + 경계 + OKX 수동 체크

포지션 보유 중 참조. 차분한 대응이 핵심.

🌐 Basis-Exit Sync (헌법 5번)

"내가 들어간 이유가 무너지면 나간다. 다른 이유로는 안 나간다."

시나리오진입 근거 = 청산 마지노선
A돌파 레벨 4H 종가 이탈
A-2MIN(진입 캔들 저점, 트리거 캔들 저점) 4H 종가 이탈 ⭐
B2회 지지 저점 4H 종가 이탈

🔴 Layer 1 — Hard Exit (전량 청산)

조건 (둘 중 하나):
□ 진입 근거 레벨 4H 종가 이탈 (시나리오별: A=돌파레벨 / A-2=MIN(진입·트리거 캔들 저점) / B=2회 지지저점)
□ Stop 발동

행동: 전량 청산 (수동 또는 자동)

🟡 Layer 2 — Critical (50% 청산)

조건 (3개 중 2개 이상):
□ 4H 음봉 거래량 5MA 1.5x 이상
□ 직전 4H 스윙 저점 이탈 (4H 종가)
□ 4H RSI < MA + RSI < 50

행동:
→ 50% 즉시 청산
→ 잔여 50%는 기존 Stop 유지 (BE 이상이면 BE)

🟢 Layer 3 — Warning (청산 ❌, 상태별 BE)

조건 (1개 충족):
□ 일봉 EMA 20 이탈 (장중)
□ 4H 윗꼬리 30% 초과 + 거래량 감소
□ 일봉 RSI 하락 다이버전스 확정 (5일 이내)

상태별 행동:

포지션 상태행동
TP0 체결 전청산 ❌ / Stop 유지 / 경계 상태
TP0 체결 후청산 ❌ / Stop을 BE(진입가)로 이동
이미 BE 이상Stop 유지 또는 더 높은 구조 Stop

📋 Layer 3 경계 체크리스트 (5건 의무)

① 4H 종가 마감까지 능동 모니터링
② 직전 4H 스윙 저점 사전 확인
③ -1.5% / -2% 추가 하락 알림 설정
④ Layer 2 트리거 추가 충족 시 즉시 50% 청산
⑤ 🔴 신규 / 2단 추가 진입 봉쇄 (핵심)

🔧 OKX 수동 체크 (4차 결함 5건 차단)

OKX는 자동으로 안 해준다. 수동 절차 의무.

□ TP0 체결 직후: SL 사이즈 자동 조정 안 됨
  → 잔여 BTC 수량 기준 SL 수동 재등록
  → 또는 2단 예약 있으면 Cancel 결정

□ TP1 주문 자동 등록 안 됨
  → 진입 직후 또는 1R+ 도달 전 수동 등록

□ BE 전환 수동
  → TP1 체결 시: SL → 진입가 (BE)
  → Layer 3 (TP0 후): SL → 진입가

□ 알림 설정 점검
  → 가격 알림 (TP1 / SL / 추가 하락 -1.5%/-2%)
  → 포지션 변경 알림 ON
  → TP0 / SL 체결 알림 필수

□ 주문 수량 일치 확인
  → SL 수량 = 잔여 BTC
  → TP0 + TP1 + 잔여 = 진입 BTC

🚫 진입 후 운영 금지 사항 (7개 중 핵심)

□ 추가 진입 ❌ (새 트리거 발생 전까지)
□ 손실 만회 사이즈 확대 ❌
□ 평단가 보정 / 물타기 ❌
□ Stop을 임의로 넓히기 ❌
□ Layer 3에서 청산 ❌ (BE 이동만)

🟦 [3-Layer]
