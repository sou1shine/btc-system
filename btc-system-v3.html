Codex #64 본질 정정 (수학적 정합 / 영구 보존):

"레버리지 3으로 나눈다고 가격 리스크가 줄어드는 것은 아닙니다. 가격 변동 손익은 notional 기준입니다. 레버리지는 증거금 요구량을 줄일 뿐, 포지션 notional의 가격 리스크를 없애지 않습니다."

v3 신 공식 (Notional 기준 / 정합):

  1. 허용 손실 USDT = 시드 × Tier 리스크% (Tier 1 = 0.5% / Tier 1.5 = 0.3% / Tier 2 = 0.5~1.0% 등급별)
  2. Stop 거리% = |진입가 - 실행 Stop| / 진입가
  3. 포지션 notional (USDT) = 허용 손실 USDT / Stop 거리%
  4. BTC 수량 = 포지션 notional / 진입가 (Lot 0.0001 BTC 내림)
  5. 증거금 검증 = notional / 3 (3x 격리) ≤ 시드 × Tier 사이즈% (notional이 격리 증거금 한도 초과 시 ❌)

예시 (시드 195.94 / Tier 1 베타 / 진입 $80,000 / Stop $79,200 = Stop 거리 1.0%):

  • 허용 손실 = 195.94 × 0.5% = $0.98
  • 포지션 notional = $0.98 / 0.01 = $98
  • BTC 수량 = $98 / $80,000 = 0.001225 → Lot 내림 = 0.0012 BTC
  • 증거금 = $98 / 3 = $32.67 ≤ 시드 × 25% (베타 상한 $48.99) ✅

본질: "레버리지 = 가격 리스크 감소" 오해 차단 / 가격 리스크 = notional 기반 / 레버리지 = 증거금 효율만
