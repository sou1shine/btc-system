🔧 gemini-mcp-tool Windows shell-quoting 패치 가이드

Windows Claude Code gemini-mcp-tool@1.1.4 @google/gemini-cli@0.42.0 · 작성: 2026-05-14 KST

한 줄 요약 — Windows에서 spawn(shell:true)이 prompt를 따옴표 없이 cmd.exe에 넘기면서 공백이 arg 구분자로 작동 → prompt가 positional arg로 분해 → gemini CLI의 신규 엄격 검증이 거부.

1. 증상

Claude Code에서 mcp__gemini__ask-gemini (또는 brainstorm) 호출 시 다음 에러:

Error executing ask-gemini: Command failed with exit code 1:
Cannot use both a positional prompt and the --prompt (-p) flag together

증상 환경:

  • OS: Windows
  • gemini-mcp-tool 글로벌 설치 (npm i -g gemini-mcp-tool)
  • @google/gemini-cli 글로벌 설치 (특정 버전부터 positional+flag 동시 사용 거부 시작)
  • 모든 prompt에서 재현 (공백 포함 prompt = 100% 실패)

2. ⚡ 빠른 자동 패치 (PowerShell)

실행 전제 — gemini-mcp-tool이 글로벌 설치되어 있어야 함. 미설치 시 먼저 npm i -g gemini-mcp-tool 실행.

PowerShell 창에서 아래 스크립트 전체 복사 → 붙여넣기 → Enter:

$ErrorActionPreference = 'Stop'

# 1. gemini-mcp-tool 경로 찾기
$npmRoot = (& npm root -g).Trim()
$target  = Join-Path $npmRoot 'gemini-mcp-tool\dist\utils\geminiExecutor.js'

if (-not (Test-Path $target)) {
    Write-Host "[ERROR] not found: $target" -ForegroundColor Red
    Write-Host "       gemini-mcp-tool 미설치. 먼저 'npm i -g gemini-mcp-tool' 실행" -ForegroundColor Yellow
    return
}

# 2. 백업 (이미 있으면 skip)
$bak = "$target.bak"
if (-not (Test-Path $bak)) {
    Copy-Item $target $bak
    Write-Host "[OK]   백업 생성: $bak" -ForegroundColor Green
} else {
    Write-Host "[INFO] 기존 백업 존재: $bak" -ForegroundColor Gray
}

# 3. 파일 읽기 (line ending 보존)
$rawContent = [System.IO.File]::ReadAllText($target)
$useCrlf    = $rawContent.Contains("`r`n")
$content    = $rawContent -replace "`r`n", "`n"

# 4. 이미 패치되었는지 확인
if ($content -match "process\.platform === 'win32'") {
    Write-Host "[SKIP] 이미 패치 적용됨" -ForegroundColor Yellow
    return
}

# 5. 패치 대상 1: 메인 경로
$old1 = @'
    // Ensure @ symbols work cross-platform by wrapping in quotes if needed
    const finalPrompt = prompt_processed.includes('@') && !prompt_processed.startsWith('"')
        ? `"${prompt_processed}"`
        : prompt_processed;
'@ -replace "`r`n", "`n"

$new1 = @'
    // Always wrap in quotes on Windows because executeCommand uses shell:true,
    // which joins args via cmd.exe and would otherwise split a prompt with
    // spaces into positional args (triggering the gemini CLI's
    // "positional prompt + -p flag" conflict). On non-Windows, only quote when
    // needed for @ syntax.
    const needsQuoting = (process.platform === 'win32' || prompt_processed.includes('@'))
        && !prompt_processed.startsWith('"');
    const finalPrompt = needsQuoting
        ? `"${prompt_processed.replace(/"/g, '\\"')}"`
        : prompt_processed;
'@ -replace "`r`n", "`n"

# 6. 패치 대상 2: Flash fallback 경로
$old2 = @'
            // Same @ symbol handling for fallback
            const fallbackPrompt = prompt_processed.includes('@') && !prompt_processed.startsWith('"')
                ? `"${prompt_processed}"`
                : prompt_processed;
'@ -replace "`r`n", "`n"

$new2 = @'
            // Same Windows shell-quoting fix for fallback (see main path above)
            const fallbackNeedsQuoting = (process.platform === 'win32' || prompt_processed.includes('@'))
                && !prompt_processed.startsWith('"');
            const fallbackPrompt = fallbackNeedsQuoting
                ? `"${prompt_processed.replace(/"/g, '\\"')}"`
                : prompt_processed;
'@ -replace "`r`n", "`n"

# 7. 패턴 존재 확인
if (-not $content.Contains($old1)) {
    Write-Host "[ERROR] 메인 패치 패턴 미발견 (gemini-mcp-tool 버전 변경 가능성)" -ForegroundColor Red
    Write-Host "        수동 패치 섹션 참조" -ForegroundColor Yellow
    return
}

# 8. 적용
$patched = $content.Replace($old1, $new1).Replace($old2, $new2)

if ($patched -eq $content) {
    Write-Host "[ERROR] 패치 미적용 (내용 변경 없음)" -ForegroundColor Red
    return
}

# 9. 쓰기 (원본 line ending 유지 / UTF-8 BOM 없이)
$final = if ($useCrlf) { $patched.Replace("`n", "`r`n") } else { $patched }
[System.IO.File]::WriteAllText($target, $final, [System.Text.UTF8Encoding]::new($false))

Write-Host ""
Write-Host "[OK]   패치 완료: $target" -ForegroundColor Green
Write-Host "[NEXT] Claude Code 완전 종료 후 재시작 → ask-gemini MCP 호출로 검증" -ForegroundColor Cyan
성공 출력 — [OK] 패치 완료: ...\geminiExecutor.js + [NEXT] Claude Code 완전 종료 후 재시작

6. 🔍 근본 원인 (3박자 결합)

#요소단독으로는결합 시
1Windows의 .cmd shim 시스템정상 (Node가 shell:true 강제)cmd.exe가 따옴표 없는 prompt를 공백으로 분해 → gemini CLI에 -p 값 + positional args 동시 전달 → "positional + flag 충돌" 에러
2gemini-mcp-tool의 @ only quotingPOSIX shell:false에선 무문제
3gemini CLI 신규 엄격 검증이전엔 동시 사용 허용 (한 쪽 우선)

코드 레벨 분해

버그 라인 (geminiExecutor.js 패치 전):

const finalPrompt = prompt_processed.includes('@') && !prompt_processed.startsWith('"')
    ? `"${prompt_processed}"`     // ← '@' 포함 시만 quoting
    : prompt_processed;            // ← 일반 prompt = quoting 없음
args.push(CLI.FLAGS.PROMPT, finalPrompt);   // → spawn args = ["-p", "Hi Gemini Hello"]

spawn 시 (commandExecutor.js):

const childProcess = spawn(command, args, {
    shell: process.platform === "win32",   // ← Windows = 강제 shell:true
    ...
});

cmd.exe 경유 변환:

cmd.exe /d /s /c "gemini -p Hi Gemini Hello"
                              ↑     ↑      ↑
                             value  pos[0] pos[1]   ← 공백으로 split
→ gemini CLI: "positional + -p 동시 사용 ❌" 거부

왜 POSIX (Linux/Mac)에선 무증상이었나

  • POSIX에서 gemini = 실제 ELF 바이너리 → spawn이 shell:false로 직접 실행 가능
  • args 배열 = OS의 execve에 그대로 전달 = 따옴표 불필요
  • = 같은 코드라도 POSIX에선 무증상 / Windows에서만 발현

7. 📌 주의사항 + 참고

  • 업스트림 PR 권장 — 패치 휘발 방지의 영구 해결책. Repo: github.com/jamubc/gemini-mcp-tool
  • 버전 종속성 — 본 패치는 gemini-mcp-tool@1.1.4 기준. 향후 버전에서 코드 라인이 바뀌면 PowerShell $old1.Contains 체크 실패 → 수동 패치 섹션의 코드 블록을 텍스트 검색으로 찾아 적용.
  • 다른 OS — 본 패치는 Windows에만 영향 (process.platform === 'win32' 분기로 격리). Linux/Mac 동작은 변경 없음.
